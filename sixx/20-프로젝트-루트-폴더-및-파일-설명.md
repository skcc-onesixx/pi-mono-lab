# 20. pi-mono-lab 프로젝트 루트 폴더 및 파일 설명

packages 외의 폴더와 파일이 어떤 역할을 하는지 정리한 문서입니다. TypeScript/코딩 지식이 없어도 이해할 수 있도록 설명합니다.

---

## 1. 전체 구조 한눈에 보기

```
pi-mono-lab/
├── packages/          ← 실제 소스 코드 (이전 문서에서 다룸)
├── .github/           ← GitHub 관련 설정 (CI, 이슈 템플릿 등)
├── .git/              ← Git 버전 관리 데이터 (직접 수정 안 함)
├── .husky/            ← Git 커밋 전 자동 검사
├── .pi/               ← 이 프로젝트용 pi 설정 (확장, 프롬프트 등)
├── scripts/           ← 빌드, 릴리스 등 유틸리티 스크립트
├── sixx/              ← 분석 문서 보관 폴더 (우리가 만든 것)
├── package.json       ← 프로젝트 메타정보, 명령어 정의
├── tsconfig.json      ← TypeScript 컴파일 설정
├── biome.json         ← 코드 스타일/린트 규칙
├── pi-test.sh         ← 소스에서 pi 실행하는 스크립트
├── test.sh            ← 테스트 실행 스크립트
└── 기타 설정 파일들
```

---

## 2. 폴더별 설명

### `.github/`
GitHub 저장소에서 사용하는 설정 폴더입니다.

| 항목 | 설명 |
|------|------|
| `workflows/` | GitHub Actions 워크플로우. push/PR 시 자동으로 빌드·테스트·검사 실행 |
| `ISSUE_TEMPLATE/` | 이슈 작성 시 사용하는 템플릿 |
| `APPROVED_CONTRIBUTORS` | 기여자 승인 목록 (원본 프로젝트용) |

**워크플로우 예시**: `ci.yml` – main 브랜치에 push하거나 PR을 만들면 `npm ci` → `npm run build` → `npm run check` → `npm test` 순으로 자동 실행됩니다.

---

### `.husky/`
Git 훅(Hook)을 관리하는 도구입니다. **커밋 직전에** 지정한 작업을 자동 실행합니다.

| 파일 | 역할 |
|------|------|
| `pre-commit` | 커밋 전 `npm run check` 실행. 실패하면 커밋이 취소됨 |

즉, 코드 스타일·타입 검사가 통과하지 않으면 커밋이 막힙니다.

---

### `.pi/`
이 **프로젝트 디렉터리**에서 pi를 실행할 때만 적용되는 설정입니다.  
(일반적으로 사용하는 `~/.pi/agent/`와는 별개입니다.)

| 폴더 | 용도 |
|------|------|
| `extensions/` | 이 프로젝트에서만 쓰는 확장 |
| `prompts/` | 이 프로젝트용 프롬프트 템플릿 |
| `git/` | `pi install git:...` 로 설치한 패키지 |
| `npm/` | `pi install npm:...` 로 설치한 패키지 |

---

### `scripts/`
프로젝트 관리용 스크립트들입니다.

| 파일 | 역할 |
|------|------|
| `sync-versions.js` | 모든 패키지 버전을 동일하게 맞춤 (lockstep 버전 관리) |
| `release.mjs` | 버전 올리기, CHANGELOG 정리, npm 배포 등 릴리스 자동화 |
| `build-binaries.sh` | pi 실행 파일(바이너리) 빌드 |
| `cost.ts` | 세션 비용 계산 유틸 |
| `session-transcripts.ts` | 세션 대화 내용 추출/변환 유틸 |

---

### `sixx/`
분석 문서를 모아두는 폴더입니다.  
- `pi-mono-lab-프로젝트-구조-분석.md` – packages 구조 분석  
- `프로젝트-루트-폴더-및-파일-설명.md` – 이 문서

## 3. 주요 파일 설명

### `package.json`
프로젝트의 **설정 파일**입니다. Node.js/npm 프로젝트에서 거의 항상 사용합니다.

| 항목 | 의미 |
|------|------|
| `name` | 프로젝트 이름 (`pi-monorepo`) |
| `version` | 버전 번호 |
| `workspaces` | 모노레포에서 포함할 패키지 경로 |
| `scripts` | `npm run build`, `npm run check` 같은 명령어 정의 |
| `dependencies` | 실행에 필요한 패키지 |
| `devDependencies` | 개발·빌드에만 필요한 패키지 |

**주요 스크립트**:
- `npm run build` – 전체 빌드
- `npm run check` – 린트, 포맷, 타입 검사
- `npm test` – 테스트 실행
- `./pi-test.sh` – 소스 코드에서 pi 실행

---

### `tsconfig.json` / `tsconfig.base.json`
**TypeScript** 컴파일 설정입니다.

- `tsconfig.base.json`: 공통 설정 (ES 버전, 모듈 방식 등)
- `tsconfig.json`: 루트 설정. `paths`로 `@mariozechner/pi-ai` 같은 이름을 실제 `packages/ai/src/` 경로에 연결

즉, "이 이름은 이 폴더를 가리킨다"는 매핑을 정의합니다.

---

### `biome.json`
**Biome**이라는 도구의 설정입니다. 코드 포맷팅과 린트 규칙을 정의합니다.

- 들여쓰기: 탭, 3칸
- 한 줄 최대 길이: 120자
- 적용 대상: `packages/*/src/**/*.ts` 등

---

### `pi-test.sh`
소스 코드를 **바로 실행**할 때 사용하는 스크립트입니다.

- `npx tsx`로 TypeScript를 실행
- `packages/coding-agent/src/cli.ts`를 진입점으로 사용
- `--no-env` 옵션: API 키 등 환경 변수를 비우고 실행 (테스트용)

**사용 예**: `./pi-test.sh` 또는 `./pi-test.sh "파일 목록 보여줘"`

---

### `test.sh`
**테스트**를 실행할 때 사용합니다.

- `auth.json`을 임시로 백업하고, API 키를 비운 뒤 테스트 실행
- LLM API가 필요한 테스트는 건너뜀 (`PI_NO_LOCAL_LLM=1`)

---

### `pi-mono.code-workspace`
**VS Code / Cursor** 워크스페이스 설정입니다.

- `pi-mono-lab`와 `moms` 폴더를 하나의 워크스페이스로 묶어서 열 수 있게 함

---

## 4. 숨김 파일/폴더 (`.` 으로 시작)

| 파일/폴더 | 설명 |
|-----------|------|
| `.git/` | Git 버전 관리 데이터. 직접 수정하지 않음 |
| `.gitignore` | Git이 추적하지 않을 파일/폴더 목록 (node_modules, dist, .env 등) |
| `.gitattributes` | 줄바꿈(LF/CRLF) 등 텍스트/바이너리 처리 방식 |
