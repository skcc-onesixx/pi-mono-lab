# 40-01. AGENTS.md: 전역 vs 프로젝트 차이

---

## 1. 수정 순서에 대해

**AGENTS.md부터 수정하는 것이 좋습니다.**

- 설정 파일(models.json, settings.json)은 **동작 방식**을 바꿈
- AGENTS.md는 **에이전트가 따를 규칙**을 정의함
- 규칙을 먼저 정해 두면, 이후 프롬프트·스킬·확장을 추가할 때 일관성이 생김

---

## 2. 전역 vs 프로젝트

| 구분 | 전역 | 프로젝트 |
|------|------|----------|
| **위치** | `~/.pi/agent/AGENTS.md` | 프로젝트 루트 등 `AGENTS.md` 또는 `CLAUDE.md` |
| **적용 범위** | 모든 프로젝트 | 해당 디렉터리 및 하위 |
| **용도** | 공통 규칙, 기본 스타일 | 프로젝트별 규칙, 기술 스택, 명령어 |

---

## 3. 로드 순서

pi는 다음 순서로 AGENTS.md를 찾아 **모두 이어 붙여** 사용합니다.

1. **전역**: `~/.pi/agent/AGENTS.md` (또는 `CLAUDE.md`)
2. **루트 → cwd**: 파일시스템 루트부터 현재 작업 디렉터리까지, 상위 → 하위 순으로 탐색
   - 예: `/AGENTS.md` → `/home/user/AGENTS.md` → `/home/user/projects/AGENTS.md` → `/home/user/projects/my-app/AGENTS.md`

**결과**: 전역 내용이 먼저, 그 다음 프로젝트(또는 상위) 내용이 이어져 하나의 컨텍스트가 됩니다.

---

## 4. 사용 전략

| 상황 | 권장 |
|------|------|
| 모든 프로젝트에 공통 규칙 | 전역 `~/.pi/agent/AGENTS.md` |
| 특정 프로젝트만 다른 규칙 | 해당 프로젝트 루트 `AGENTS.md` |
| 팀/회사 공통 규칙 + 프로젝트별 규칙 | 전역에 공통, 프로젝트에 개별 규칙 |

---

## 5. 예시 파일 위치

- **전역 예시**: `sixx/40-02-AGENTS-예시-전역.md`
- **프로젝트 예시**: `sixx/40-03-AGENTS-예시-프로젝트.md`

위 예시를 참고해 `~/.pi/agent/AGENTS.md` 또는 프로젝트 루트 `AGENTS.md`를 만들면 됩니다.
