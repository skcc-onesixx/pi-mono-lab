# 30. pi 에이전트 커스터마이징 가이드

Ollama를 다른 서버에 두고 pi를 사용 중일 때, 에이전트를 의도에 맞게 수정하는 방법을 정리한 문서입니다.

---

## 1. 현재 상황 요약

- **상황**: pi가 다른 서버에 설치되어 있고, Ollama 모델과 연결됨
- **목표**: 에이전트를 조금씩 수정해서 의도에 맞게 맞추기

---

## 2. 커스터마이징 난이도별 정리

pi는 **소스 코드를 직접 수정하지 않고**도 여러 방법으로 커스터마이징할 수 있습니다.

| 난이도 | 방법 | 설명 | 코드 필요 |
|--------|------|------|----------|
| **쉬움** | 설정 파일 | models.json, settings.json | 없음 |
| **쉬움** | AGENTS.md | 프로젝트별 지시사항 | 마크다운 |
| **쉬움** | SYSTEM.md | 시스템 프롬프트 변경 | 마크다운 |
| **쉬움** | 프롬프트 템플릿 | `/이름`으로 재사용 가능한 프롬프트 | 마크다운 |
| **쉬움** | 스킬(Skills) | 작업별 가이드, 모델이 필요 시 로드 | 마크다운 |
| **중간** | 확장(Extensions) | 도구, 명령, UI, 이벤트 처리 | TypeScript |
| **어려움** | 소스 직접 수정 | fork 후 수정 | TypeScript |

---

## 3. 다른 서버의 Ollama 연결

Ollama가 `http://서버주소:11434` 같은 다른 서버에 있다면, `models.json`에서 `baseUrl`만 바꾸면 됩니다.

**파일 위치**: `~/.pi/agent/models.json` (pi가 설치된 서버 기준)

```json
{
  "providers": {
    "ollama": {
      "baseUrl": "http://YOUR_SERVER_IP:11434/v1",
      "api": "openai-completions",
      "apiKey": "ollama",
      "models": [
        { "id": "llama3.1:8b" },
        { "id": "qwen2.5-coder:7b" }
      ]
    }
  }
}
```

- `baseUrl`: Ollama 서버 주소 (예: `http://192.168.1.100:11434/v1`)
- `apiKey`: Ollama는 무시하므로 `"ollama"` 등 아무 값
- `/model`에서 모델 선택 후 `/reload`로 새로고침 불필요

**참고**: `packages/coding-agent/docs/models.md`

## 4. 커스터마이징 방법별 상세

### 4.1 프로젝트별 지시사항 (AGENTS.md)

**용도**: 프로젝트마다 다른 규칙, 컨벤션, 공통 명령어 지정

**위치**:
- 전역: `~/.pi/agent/AGENTS.md`
- 프로젝트: `.pi/AGENTS.md` 또는 프로젝트 루트 `AGENTS.md`

**내용 예시** (마크다운):

```markdown
# 프로젝트 규칙

- 항상 Python 3.11 이상 사용
- 코드 스타일: Black 포맷터 사용
- 테스트는 pytest로 실행
```

**참고**: `packages/coding-agent/README.md`의 "Context Files" 섹션

---

### 4.2 시스템 프롬프트 변경 (SYSTEM.md)

**용도**: 에이전트 기본 성격·역할·행동 방식 변경

**위치**:
- 전역: `~/.pi/agent/SYSTEM.md`
- 프로젝트: `.pi/SYSTEM.md` (프로젝트 우선)

**추가만 하려면**: `APPEND_SYSTEM.md` 사용

**내용 예시**:

```markdown
당신은 내부 개발용 코딩 어시스턴트입니다.
- 응답은 한국어로 간결하게 작성
- 파일 수정 전 반드시 사용자 확인
```

**참고**: `packages/coding-agent/README.md`의 "Context Files" 섹션

---

### 4.3 프롬프트 템플릿

**용도**: 자주 쓰는 프롬프트를 `/이름`으로 재사용

**위치**:
- 전역: `~/.pi/agent/prompts/*.md`
- 프로젝트: `.pi/prompts/*.md`

**예시** (`~/.pi/agent/prompts/리뷰.md`):

```markdown
---
description: 코드 리뷰 요청
---
다음 코드를 리뷰해 주세요. 버그, 보안, 성능 관점에서 검토해 주세요.
```

에디터에서 `/리뷰` 입력 시 위 내용이 자동으로 채워집니다.

**참고**: `packages/coding-agent/docs/prompt-templates.md`

---

### 4.4 스킬(Skills)

**용도**: 특정 작업(예: 배포, 문서화)에 대한 단계별 가이드

**위치**:
- 전역: `~/.pi/agent/skills/`, `~/.agents/skills/`
- 프로젝트: `.pi/skills/`, `.agents/skills/`

**예시** (`~/.pi/agent/skills/배포/SKILL.md`):

```markdown
# 배포 스킬

배포 프로세스 실행 시 사용.

## Steps
1. `npm run build` 실행
2. `./deploy.sh` 실행
3. 결과 확인
```

**사용**: `/skill:배포` 또는 모델이 자동으로 필요 시 로드

**참고**: `packages/coding-agent/docs/skills.md`

---

### 4.5 설정 (settings.json)

**용도**: 기본 모델, 테마, 컴팩션, 재시도 등 동작 설정

**위치**:
- 전역: `~/.pi/agent/settings.json`
- 프로젝트: `.pi/settings.json`

**예시**:

```json
{
  "defaultProvider": "ollama",
  "defaultModel": "qwen2.5-coder:7b",
  "theme": "dark",
  "compaction": {
    "enabled": true,
    "reserveTokens": 16384
  }
}
```

**참고**: `packages/coding-agent/docs/settings.md`

---

### 4.6 확장(Extensions)

**용도**: 도구 추가, 명령 추가, 도구 호출 차단, UI 변경 등

**위치**:
- 전역: `~/.pi/agent/extensions/*.ts`
- 프로젝트: `.pi/extensions/*.ts`

**예시** (`~/.pi/agent/extensions/위험-확인.ts`):

```typescript
import type { ExtensionAPI } from "@mariozechner/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.on("tool_call", async (event, ctx) => {
    if (event.toolName === "bash" && event.input.command?.includes("rm -rf")) {
      const ok = await ctx.ui.confirm("위험!", "rm -rf 실행을 허용할까요?");
      if (!ok) return { block: true, reason: "사용자가 차단함" };
    }
  });
}
```

**테스트**: `pi -e ./위험-확인.ts`

**참고**: `packages/coding-agent/docs/extensions.md`, `packages/coding-agent/examples/extensions/`

---

### 4.7 커스텀 프로바이더 (Extension)

**용도**: Ollama 외 API, OAuth, 비표준 API 등

**예시**: `packages/coding-agent/examples/extensions/custom-provider-anthropic/`

**참고**: `packages/coding-agent/docs/custom-provider.md`

## 5. 추천 순서

1. **Ollama 연결 확인**: `models.json`의 `baseUrl`이 올바른지 확인
2. **AGENTS.md**: 프로젝트별 규칙·컨벤션 정의
3. **SYSTEM.md**: 에이전트 기본 성격·역할 정의
4. **settings.json**: 기본 모델, 테마 등 설정
5. **프롬프트 템플릿**: 자주 쓰는 프롬프트 등록
6. **스킬**: 반복 작업용 가이드 정의
7. **확장**: 도구/명령/UI 차단 등 고급 기능

## 7. 소스 코드 직접 수정이 필요한 경우

설정/확장으로 해결이 안 되면, 이 fork를 수정해 사용할 수 있습니다.

**포크 설정 예시** (`packages/coding-agent/package.json`):

```json
{
  "piConfig": {
    "name": "pi",
    "configDir": ".pi"
  }
}
```

- `name`: CLI 이름
- `configDir`: 설정 디렉터리 이름

**소스 실행**: `./pi-test.sh` (프로젝트 루트에서)

**참고**: `packages/coding-agent/docs/development.md`
